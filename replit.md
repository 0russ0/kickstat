# Kicking Tracker

A mobile-first app for parents to track their athletes' field goals, punts, and kickoffs with real-time cloud sync.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/mobile run dev` — run the Expo mobile app
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string (auto-provisioned)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Mobile: Expo (React Native) with Expo Router
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth)
- `lib/db/src/schema/athletes.ts` — athletes table
- `lib/db/src/schema/kicks.ts` — kicks table (JSONB data column)
- `artifacts/api-server/src/routes/athletes.ts` — athlete CRUD routes
- `artifacts/api-server/src/routes/kicks.ts` — kick tracking routes
- `artifacts/mobile/app/(tabs)/` — screens: index (FG), punt, kickoff, history
- `artifacts/mobile/context/AppContext.tsx` — shared state (athletes, sync status)
- `artifacts/mobile/hooks/useStopwatch.ts` — high-precision stopwatch
- `artifacts/mobile/constants/colors.ts` — Slate-950/Blue-600 palette

## Architecture decisions

- Kick data is stored as JSONB to accommodate the three different kick types (field_goal, punt, kickoff) without separate tables
- Max 3 athletes enforced at the API level
- Cloud Sync indicator uses Reanimated pulse animation tied to API mutation lifecycle
- `performance.now()` used in the stopwatch hook for sub-millisecond accuracy; `onPressIn` on native for zero-latency start
- Orval generates both React Query hooks (`lib/api-client-react`) and Zod validators (`lib/api-zod`) from the single OpenAPI spec

## Product

- **Athletes**: Up to 3 athletes selectable via the persistent top bar
- **Field Goal**: Enter LOS → auto-calculates total distance (LOS+10); record Made or Missed (Left/Right/Short/Blocked/Bad Snap)
- **Punt**: Enter snap yard + landing yard → auto-calculates distance; high-precision hangtime stopwatch
- **Kickoff**: Touchback toggle (Endzone/Out of Endzone), hangtime stopwatch
- **History**: Last 10 kicks per screen with delete; season stats on the History tab
- **Cloud Sync**: Pulsing amber dot while writes are in progress, green when synced

## Gotchas

- Run `pnpm run typecheck:libs` after any changes to `lib/api-zod/src/index.ts` — it gets regenerated by Orval and the output needs to only re-export from `./generated/api` (not `./generated/types`)
- Always run codegen (`pnpm --filter @workspace/api-spec run codegen`) after OpenAPI spec changes, then manually set `lib/api-zod/src/index.ts` to `export * from "./generated/api"` only
